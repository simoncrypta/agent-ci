# Digital Twin Universe (DTU) Setup

## Summary

We implemented a "Digital Twin Universe" (DTU) to simulate GitHub Actions locally. This allows for end-to-end testing of the `machinen-bridge` and `machinen-runner` without needing real GitHub interaction. We successfully created mock payloads, a simulation script with HMAC signing, and updated both the Bridge and Runner to support this local sandbox.

## The Problem

We needed a way to test features end-to-end (webhook -> bridge -> runner -> executor) without invoking GitHub. This required:

1.  Mcking GitHub webhook events.
2.  Handling GitHub's HMAC signature verification in the Bridge.
3.  Mocking the GitHub API endpoint that the Runner uses to fetch job details.

## Investigation & Timeline

- **Initial State:** The Machinen Bridge was filtering for `workflow_job` events and verifying signatures. The Runner was hardcoded to use `api.github.com`.
- **Attempts:**
  - Created `dtu/github-actions` for simulation tools.
  - Drafted mock payloads for `push` and `workflow_job`.
  - Developed `simulate.ts` to sign payloads and POST to the Bridge.
  - Discovered the Runner needed a custom `GITHUB_API_URL` to point to the Bridge instead of real GitHub.
  - Added a mock route in the Bridge (`/repos/:owner/:repo/actions/jobs/:jobId`) to serve the stored webhook payload as "job details".

## Discovery & Key Findings

- The Bridge was strictly filtering for `workflow_job`. Testing with `push` payloads was ignored.
- The Bridge requires `X-Hub-Signature-256`, so the simulation script must implement HMAC-SHA256 signing.
- The Runner's `executor` pulls its own spec via `curl` against the GitHub API. By making `GITHUB_API_URL` configurable and mocking that endpoint in the Bridge, we achieved a closed loop.

## Resolution

- **[DTU]**: Created `dtu/github-actions/` with `simulate.ts`, `README.md`, and JSON events.
- **[Runner]**: Added `GITHUB_API_URL` to `config.ts` and updated `executor.ts` to use it.
- **[Bridge]**: Added `handleGitHubJobDetails` route and job ID mapping in `api/routes.ts`.
